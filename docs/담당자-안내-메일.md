**제목:** [사내 코파일럿 경진대회] U+ VOICE · VOC Action Copilot — 에이전트 2종 + 운영 사이트 안내

안녕하세요, 담당자님.

사내 Microsoft 365 Copilot 활용 경진대회 출품작 **‘U+ VOICE · VOC Action Copilot’** 을 공유드립니다.
고객의 소리(VOC)를 코파일럿으로 **분류·응대 생성**하고, 사내 운영 사이트에서 **티켓처럼 처리**한 뒤 지라로 연동하는 CX 코파일럿입니다.

**■ 구성**
- **에이전트 2종** (M365 Copilot **에이전트 빌더 · 선언적 에이전트**, Copilot Studio 아님)
  1. `U+VOC 고객가이드` — VOC를 6그룹·22표준분류·대응영역으로 분류하고 요약·예상 응대문·개선 과제 생성
  2. `U+VOC 셀프가이드` — 같은 분류 지식으로 고객 **셀프 해결 단계 + 선제 안내문(이메일·문자)** 생성, 미해결 건만 상담 연결
- **운영 사이트** `U+ VOICE` — VOC 보드 · VOC 처리(지라형 티켓) · 셀프 해결 가이드 · 인사이트 · Copilot Agent 연동

**■ 닫힌 루프**
코파일럿이 **생성** → 담당자가 **검수** → 사이트에서 **처리·노출** → 사내 지라로 **연동** → 처리결과를 분류 모델로 **재학습**.
(고객이 에이전트와 직접 대화하지 않고, 담당자가 코파일럿으로 생성·검수해 사이트가 고객에 노출하는 구조입니다.)

**■ 정확도 핵심 — 서비스 이해 지식**
에이전트가 VOC를 정확히 이해하려면 U+one 앱·홈페이지 **서비스 구조·용어(요금제·결합·VIP콕·유독·이심·로밍 등)** 를 알아야 합니다.
이를 위해 **서비스 이해 지식 문서**를 Knowledge 1순위로 넣고, **www.uplus.co.kr 을 웹 지식 소스로 추가**하며, 모르는 내용은 추측하지 않고 ‘확인 필요’로 답하도록 지시문에 명시했습니다.
