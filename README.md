# U+ VOC Action Copilot (공모전 MVP · 사내 전용)

React + Vite 정적 SPA. VOC를 수집·자동분류하고 선제조치를 돕는 데모.

## 실행

```bash
npm install
npm run dev      # 개발
npm run build    # 배포 빌드 (dist/)
```

배포는 Vercel에 `git push` (정적 빌드).

## 데이터 모드

앱은 두 가지 모드로 동작합니다. 환경변수 유무로 자동 결정돼요.

### 1) 로컬 모드 (기본 · 환경변수 없음)
- 입력 데이터는 **각 브라우저 localStorage**에 저장됩니다.
- 내 브라우저에 데이터가 없으면 배포본의 **`public/seed.json`** 을 자동으로 불러옵니다
  → 로그인한 누구나 같은 "기본 데이터"를 봅니다(스냅샷).
- 내 실제 데이터를 모두에게 보이려면: **VOC 수집·입력 → 현재 데이터 내보내기(seed.json)**
  로 받아 `public/seed.json`을 교체하고 배포(git push).
- 한계: 실시간 누적은 안 됩니다(각자 입력은 그 브라우저에만 남음).

### 2) 공유 저장소 모드 (실시간 누적 · 권장)
- `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` 를 설정하면 활성화.
- 입력·붙여넣은 VOC가 **공유 저장소에 적재**되고, 약 4초 폴링으로
  **로그인한 모든 사용자에게 누적 표시**됩니다. 상단에 "공유 저장소 연결됨" 배지.
- 설정 방법은 **`.env.example`** 참고:
  1. Supabase 프로젝트 생성(무료)
  2. SQL Editor에서 테이블 생성 + (데모용) RLS 해제
  3. 로컬은 `.env`(또는 `.env.local`), Vercel은 **Settings → Environment Variables** 에 두 값 등록 후 재배포
- 데모용 편의: **VOC 수집·입력 → "공유 데이터에 샘플 시드 넣기"** 로 seed.json을
  공유 저장소에 한 번에 적재할 수 있습니다("공유 데이터 비우기"로 초기화).

> 참고: 폴링은 일반 HTTPS GET/POST만 사용해 웹소켓보다 제한적 사내망 통과 가능성이 높습니다.
> 외부망에서 먼저 동작을 보여주고, 사내에서는 동일 패턴(허용된 사내 저장소/엔드포인트)으로 적용하면 됩니다.
> anon key는 브라우저에 노출되는 공개 키입니다. 민감 정보는 저장하지 마세요.
